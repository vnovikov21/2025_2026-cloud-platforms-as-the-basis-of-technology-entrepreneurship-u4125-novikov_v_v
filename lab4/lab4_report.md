```markdown
## Описание приложения
Приложение представляет собой **AI-ассистента** (интеллектуальный чат-бот), который помогает пользователям генерировать тексты, идеи, анализировать данные и работать с изображениями с помощью современных AI-моделей (Gemini / Vertex AI).

**Ожидаемая нагрузка:**
- **Initial (MVP)**: до 100 активных пользователей в сутки
- **Testing**: до 1000 активных пользователей в сутки
- **Production**: от 5000+ активных пользователей в сутки
```

**Блок 4 — Экономическая модель**

```markdown
## Экономическая модель

| Стадия              | Основные сервисы                              | Примерная стоимость в месяц | Обоснование |
|---------------------|-----------------------------------------------|-----------------------------|-----------|
| Initial (MVP)       | Cloud Run + Firestore + Gemini API            | 1 500 – 3 000 ₽            | Минимальные затраты, быстрое тестирование |
| Testing             | Cloud Run + Cloud SQL + Load Balancer         | 5 000 – 9 000 ₽            | Надёжность и A/B-тестирование версий |
| Production          | Полный стек + CDN + Redis                     | 30 000 – 60 000 ₽          | Высокая нагрузка, стабильность и масштабируемость |

*Цены ориентировочные (зависят от региона и фактического использования).
```

**Блок 5 — Обоснование и Выводы**

```markdown
## Обоснование выбора сервисов

- **Cloud Run** — serverless контейнеры, удобно и дёшево на старте.
- **Firestore** → **Cloud SQL** — постепенный переход от простой NoSQL к полноценной реляционной БД.
- **Vertex AI / Gemini** — мощные AI-модели от Google с удобной интеграцией.
- **Load Balancer + CDN** — распределение нагрузки и ускорение работы для пользователей.
- **Redis (Memorystore)** — кэширование для высокой производительности на продакшене.

## Выводы

В ходе выполнения лабораторной работы была спроектирована инфраструктура AI-приложения для трёх этапов развития.

Главный вывод: инфраструктуру нужно развивать **поэтапно**. Начинать следует с минимально возможного и недорогого решения (MVP), чтобы быстро проверить гипотезу. По мере роста нагрузки и обратной связи от пользователей добавлять более сложные и надёжные компоненты.

Такой подход позволяет значительно экономить средства на старте, быстро итератировать продукт и минимизировать риски при масштабировании.
```
