## Цель работы
Ознакомиться с облачным объектным хранилищем Google Cloud Storage, научиться создавать бакеты, загружать файлы, организовывать их в папки, настраивать доступ и получать публичные ссылки.

## Ход работы

1. **Создание Bucket**  
   В проекте был создан бакет `vnovikov-lab3-bucket` (регион europe-west1, класс хранения Standard).

2. **Загрузка файлов**  
   В бакет были загружены 4 изображения.  
   (Скриншот 1 — список загруженных файлов)

3. **Создание папки и перемещение**  
   Была создана папка `images`.  
   Все изображения перемещены внутрь этой папки для удобной организации.  
   (Скриншот 2 — структура с папкой)

4. **Настройка публичного доступа**  
   Для всех файлов была настроена публичная доступность через роль `allUsers` + `Storage Object Viewer`.  
   (Скриншот 3 — настройки доступа)

5. **Создание публичных ссылок**  
   Для каждого файла была получена публичная ссылка.  
   По этим ссылкам изображения успешно открываются в браузере даже в режиме инкогнито.  
   (Скриншоты 4-5 — примеры публичных ссылок и открытые изображения)

6. **Удаление ресурсов**  
   После выполнения работы бакет был полностью удалён.

## Выводы

В ходе лабораторной работы я познакомился с объектным хранилищем Cloud Storage.  
Основные выводы:

- Cloud Storage очень удобен для хранения файлов, фотографий, документов и других объектов.
- Организация данных через папки помогает поддерживать порядок.
- Публичный доступ настраивается очень просто и позволяет легко делиться файлами через прямые ссылки.
- Важно понимать разницу между разными классами хранения (Standard, Nearline, Coldline и т.д.) — они влияют на цену и частоту доступа.
- Как и в предыдущих лабораторных, после работы обязательно нужно удалять созданные ресурсы, чтобы не расходовать квоты проекта.

Cloud Storage отлично подходит для статических файлов, бэкапов, медиа-контента и интеграции с другими сервисами Google Cloud (например, с Cloud Run или виртуальными машинами).
