## Цель работы
Ознакомиться с сервисом Google Cloud Run, научиться разворачивать контейнеры, работать с ревизиями, анализировать логи и метрики, а также управлять трафиком между версиями.

## Ход работы

1. **Создание сервиса**  
   В Google Cloud Console я создал новый сервис Cloud Run с названием `vnovikov-cloudrun-lab2`.  
   Использовал готовый образ `us-docker.pkg.dev/cloudrun/container/hello`.  
   Установил минимальные ресурсы: 0.5 vCPU и 256 MiB памяти.

2. **Тестирование**  
   После создания сервиса получил публичную ссылку. При переходе по ней отобразилась страница "Hello World!".  
   (Скриншот 1 — главная страница сервиса)

3. **Анализ логов и метрик**  
   Перешёл во вкладки **Logs** и **Metrics**.  
   В логах отображались все запросы к сервису.  
   В метриках можно видеть количество запросов, использование CPU и памяти.  
   (Скриншоты 2-3 — Logs и Metrics)

4. **Изменение конфигурации**  
   Отредактировал сервис и поменял Container port с 8080 на 8090.  
   После деплоя новой ревизии сервис перестал работать корректно (ошибка 502).  
   Это произошло потому, что приложение внутри контейнера ожидает запросы на порту 8080.  
   (Скриншот 4 — ошибка после смены порта)

5. **Управление трафиком**  
   Через раздел **Manage Traffic** переключал процент трафика между старой (рабочая) и новой (неработающая) ревизиями.  
   При split 50/50 при обновлении страницы результат был разный.  
   (Скриншоты 5-6 — управление трафиком и результат)

6. **Удаление ресурсов**  
   После завершения работы полностью удалил Cloud Run сервис.

## Выводы

Cloud Run — это удобный и простой сервис для запуска контейнеров без управления серверами.  
Главные преимущества:
- Автоматическое масштабирование (от 0 экземпляров)
- Легкое управление версиями (Revisions)
- Возможность постепенного переключения трафика (очень полезно для обновлений без downtime)

Я понял, как важно правильно указывать порт контейнера и как удобно тестировать новые версии, не ломая основной сервис. Удалил сервис после работы, чтобы не тратить ресурсы проекта.
